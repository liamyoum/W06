# C 자료구조 구현 + CSAPP 3장 학습 레포지토리

이 저장소는 크래프톤 정글 12기 6주차에 진행한 학습 내용을 정리한 레포지토리다.

- C로 자료구조를 직접 구현하면서 `struct`, 포인터, 동적 메모리 할당, 재귀 호출에 익숙해지기
- `CSAPP` 3장을 읽으면서 C 코드가 컴파일되어 어셈블리와 기계어 수준으로 내려가고, 최종적으로 컴퓨터가 실행하는 전체 흐름을 큰 그림으로 이해하기.

## 이번 주 학습 목표

- C 문법을 "읽는 수준"에서 끝내지 않고 직접 구현하면서 손에 익히기
- 연결 리스트, 스택/큐, 트리, BST를 구현하며 자료구조의 동작 원리를 코드로 확인하기
- 함수 호출, 포인터 접근, 메모리 구조가 실제 실행 단계에서 어떻게 이어지는지 감각 잡기
- `C 소스 코드 -> 컴파일 -> 어셈블리 -> 실행 파일 -> CPU 실행`의 흐름을 CSAPP 3장을 통해 큰 틀에서 이해하기

## 레포 구성

```text
data_structures_docker/
├── README.md
├── .devcontainer/                 # Dev Container 설정
├── .vscode/                       # VSCode 빌드/디버깅 설정
├── CSAPP 3장 입문편 (3.1_3.12 전체).pdf
├── CSAPP 3장 심화편 (3.1_3.12 전체).pdf
├── CSApp 공부법.pdf
├── PINTOS_전_C_기초_특강_발표자료.pdf
└── Data-Structures/
    ├── practice.c                 # 간단한 C 문법 연습 파일
    ├── Linked_List/               # 7개 문제 + 문제 PDF
    ├── Stack_and_Queue/           # 7개 문제 + 문제 PDF
    ├── Binary_Tree/               # 8개 문제 + 문제 PDF
    └── Binary_Search_Tree/        # 5개 문제 + 문제 PDF
```

총 `28개`의 C 문제 파일이 들어 있다.

각 문제 폴더에는 다음이 함께 들어 있다.

- `Q*.c`: 자료구조 문제를 푼 C 소스 코드
- `*.pdf`: 해당 주제의 문제 설명 문서
- 확장자 없는 동일 이름 파일: 컴파일 결과로 생성된 실행 파일

현재 저장된 확장자 없는 실행 파일들은 Dev Container 환경에서 생성된 Linux ELF 바이너리다.

## 학습 흐름

이 레포에서 자료구조 문제는 아래 순서로 진행하는 흐름이 자연스럽다.

1. `Linked_List`
2. `Stack_and_Queue`
3. `Binary_Tree`
4. `Binary_Search_Tree`

연결 구조를 직접 다루는 과정에서 포인터와 메모리 접근에 익숙해지고, 트리와 BST로 넘어가면서 재귀, 순회, 레벨 탐색 같은 개념을 더 분명하게 볼 수 있다.

## 코드 특징

- 대부분의 파일은 문제에서 제공된 메인 프레임 위에 필요한 함수를 구현하는 형태다.
- `main()`이 포함되어 있어서 입력을 주며 바로 동작을 확인할 수 있다.
- 연결 리스트에서는 정렬 삽입, 노드 순회, 삽입/삭제 같은 기본 조작을 연습했다.
- 스택/큐에서는 연결 리스트 기반 구현과 값 필터링 같은 조작을 다뤘다.
- 이진 트리와 BST에서는 재귀, 구조 비교, 순회, 레벨 순회 같은 문제를 풀었다.

## CSAPP 3장과 연결해서 본 포인트

자료구조 문제를 C로 직접 구현하면서, CSAPP 3장에서 보던 내용이 다음처럼 이어졌다.

- 포인터 연산과 메모리 주소 접근이 실제로는 메모리 참조 명령으로 내려간다는 점
- 함수 호출과 재귀가 스택 프레임의 생성/복구와 연결된다는 점
- `struct`와 배열 접근이 메모리 배치와 오프셋 계산으로 이어진다는 점
- 조건문, 반복문이 분기와 점프 형태로 번역된다는 점

이 주의 핵심은 어셈블리 한 줄 한 줄을 완벽히 해석하는 것보다, `C 코드가 결국 어떤 식으로 기계가 읽는 형태로 바뀌는가`를 큰 흐름 차원에서 이해하는 데 있었다.

## 실행 방법

이 레포에는 `.devcontainer`와 `.vscode` 설정이 포함되어 있어서 VSCode + Dev Container 환경에서 바로 빌드/디버깅할 수 있다.

직접 컴파일해서 실행하려면 예를 들어:

```bash
gcc -g Data-Structures/Linked_List/Q1_A_LL.c -o Data-Structures/Linked_List/Q1_A_LL
./Data-Structures/Linked_List/Q1_A_LL
```

또는 VSCode에서 원하는 `*.c` 파일을 열고 빌드/디버깅 설정을 사용할 수 있다.

- `.vscode/tasks.json`: 현재 파일 컴파일
- `.vscode/launch.json`: `gdb` 기반 디버깅
- `.devcontainer/`: GCC, GDB, Valgrind 등이 포함된 개발 환경
